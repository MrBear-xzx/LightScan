# Commit Message 规范

- 版本：v1.1
- 生效日期：2026-06-23

## 1. 格式

`<type>(<scope>): <subject>`

- `type`：变更类型（英文关键字）
- `scope`：影响范围（推荐）
- `subject`：一句话描述（**中文**，建议 <= 72 字符）

示例：
- `feat(discovery): 增加发现任务创建接口`
- `fix(dedup): 修复同指纹漏洞重复聚合问题`
- `docs(prd): 补充质量规范章节`

## 2. type 取值

- `feat`：新功能
- `fix`：缺陷修复
- `refactor`：重构（不改功能）
- `perf`：性能优化
- `test`：测试相关
- `docs`：文档变更
- `chore`：杂项维护（构建、依赖、脚手架）
- `ci`：CI/CD 变更
- `revert`：回滚提交

## 3. 编写要求

- 提交标题使用中文，描述“做了什么”。
- 不写“update/fix bug”等模糊词。
- 一个 commit 只做一件事。
- 必要时补充 body 说明原因、风险、兼容性（建议中文）。

## 4. 推荐模板

```text
<type>(<scope>): <中文标题>

背景：<为什么做>
变更：<做了什么>
风险：<可能影响>
验证：<测试与结果>
```

## 5. 示例

```text
feat(scan): 增加 nuclei 结果标准化适配器

背景：多引擎结果字段不统一，影响去重与评分。
变更：新增 nuclei 结果标准化映射和指纹生成。
风险：历史数据兼容性。
验证：新增单测并通过 pytest。
```
